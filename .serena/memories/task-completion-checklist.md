# タスク完了時の必須チェックリスト

## 絶対的な原則

**エラーがある状態でタスクを完了してはならない。ゼロエラー原則を厳守する。**

## 必須ワークフロー

### 1. コード変更後の品質チェック

```bash
pnpm lint:check   # 警告も含めて0個必須
pnpm typecheck    # TypeScriptエラー0個必須
pnpm format:check # フォーマットエラー0個必須
```

### 2. テスト実行確認

```bash
pnpm test:run     # 全テストパス必須
```

### 3. エラー修正（エラーが存在する場合）

```bash
pnpm lint         # ESLint自動修正
pnpm format       # Prettier自動修正
# 手動修正後、再度チェック実行
```

### 4. 最終確認

- 再度全品質チェックを実行
- 全てのエラー・警告が解消されていることを確認
- テストが全てパスしていることを確認

## 禁止事項

❌ **警告やエラーがある状態での作業完了**  
❌ **品質チェックをスキップした作業完了**  
❌ **テスト実行なしの作業完了**  
❌ **「後で修正」の前提でのタスク完了**

## TodoWrite統合

コード変更を含むタスクでは以下を必須項目として追加：

- "エラーチェック・解決"
- "全品質チェック実行"
- "テスト実行確認"

## t-wada式TDD要件

新機能・仕様変更時は以下プロセスを必須実行：

1. 🔴 Red: 失敗するテストを書く→テスト実行で失敗確認
2. 🟢 Green: 最小実装でテスト通す→テスト実行で成功確認
3. 🔵 Refactor: 必要に応じてリファクタリング→品質チェック
